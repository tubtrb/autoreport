# About Autoreport

Autoreport is a structured reporting product for teams that need repeatable presentation output. Instead of rebuilding the same report layout by hand every cycle, the product keeps a defined authoring contract, validates the working draft, and generates an editable PowerPoint deck.

## Who this site is for

This public site is for readers who want to understand the current hosted workflow:

- what the hosted demo does
- how to prepare a draft
- how to attach supporting screenshots
- what changed between releases

The public pages are written for hosted demo users, evaluators, and teams that want a quick product-level understanding before going deeper.

## What is published here

`auto-report.org` keeps the public product story narrow:

- the homepage explains the product and the current hosted flow
- the User Guide walks through the public browser workflow
- the Updates page groups release notes and development logs
- supporting pages explain common workflow questions in plain language

## What stays on GitHub

The open GitHub repository is the right place for:

- implementation history
- contributor-facing details
- issue tracking
- source code and test changes
- deeper technical inspection

Repository: [tubtrb/autoreport](https://github.com/tubtrb/autoreport)

## How the site is maintained

This site is maintained under the Autoreport brand and updated alongside the public product workflow. When the hosted experience changes in a user-visible way, the homepage, guide, and release-facing posts are updated to match that public behavior.
