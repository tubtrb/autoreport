# Contact

Autoreport keeps public contact lightweight and project-focused.

## Best contact paths

- Product and bug reports: [GitHub Issues](https://github.com/tubtrb/autoreport/issues)
- Repository and release history: [GitHub Repository](https://github.com/tubtrb/autoreport)
- Public updates and release context: [Updates page](/%EC%97%85%EB%8D%B0%EC%9D%B4%ED%8A%B8/)

## What to include in a report

When you report a problem with the hosted workflow, include:

- the page you opened
- the step that failed
- whether the issue happened during draft checking, preview refresh, screenshot upload, or generation
- any message that was visible in the browser

## Current support model

Autoreport does not use a public contact form or a separate support mailbox from this publishing repository today. Public feedback is handled through the project repository so product issues, release context, and fixes can stay aligned in one place.
